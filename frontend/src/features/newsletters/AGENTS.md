# newsletters

## Purpose
Newsletter list and editor. Editor fields are what an operator fills in before clicking "Run Now": name, description, prompt, AI provider + optional pinned API key, Resend API key, from email, model, template, timezone, optional cron schedule, notes, and recipient import.

## Key Files
| File | Description |
|------|-------------|
| `NewsletterListPage.tsx` | Table of newsletters with per-row action dropdown — Run Now / Edit / Pause / Resume / Pause Schedule / Resume Schedule / Archive / Delete |
| `NewsletterEditorPage.tsx` | Create / edit form. Loads `api.formOptions()` for providers, models, templates, timezones. Does NOT expose editable subject / preheader / body-text fields — those are generated by the AI at run time |
| `newsletter-types.ts` | `NewsletterSummary`, `NewsletterDetail`, `NewsletterInput`, `emptyNewsletterInput`, `toNewsletterInput` helper |

## For AI Agents

### Working In This Directory
- Do NOT reintroduce Subject / Preheader / Body Text editor fields. The AI generates those at run time. The prompt is the only content input the operator manages.
- Do NOT add draft / revision / preview / test-send buttons. The product flow is prompt → run → email sent; previous versions had all of those and they were removed on purpose.
- "Pause" is a newsletter status toggle (status → paused). "Pause Schedule" disables the cron independently. These are intentionally separate actions — don't merge them.
- `handleRunNewsletter` in `App.tsx` calls `api.runNewsletter(id)` which hits a single-step backend endpoint; successful runs include a recipient-count message in the notice banner.
